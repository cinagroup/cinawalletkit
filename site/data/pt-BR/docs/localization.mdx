# Localização

## Customizando o idioma do seu dApp com traduções integradas

Por padrão, o CinaWalletKit oferece suporte ao local `en-US` para usuários da língua inglesa.

Se disponível, o CinaWalletKit detectará a [linguagem preferida](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language) do usuário e escolherá as traduções apropriadas. Os desenvolvedores sempre podem sobrescrever o idioma padrão.

Para especificar um idioma para seus usuários, basta adicionar `locale="zh-CN"` como uma propriedade no seu `CinaWalletKitProvider`

```tsx
import { CinaWalletKitProvider } from '@cinagroup/CinaWalletKit';

export const App = () => (
  <CinaWalletKitProvider locale="zh-CN" {...etc}>
    {/* Your App */}
  </CinaWalletKitProvider>
);
```

### Usando com Next.js

O suporte à localização do CinaWalletKit funciona ainda melhor com [Sub-path Routing](https://nextjs.org/docs/pages/building-your-application/routing/internationalization#sub-path-routing).

Configure o seu projeto Next.js como o exemplo abaixo para adicionar um subcaminho `/locale/` opcional às suas rotas, o que ajudará os motores de busca e os usuários a descobrirem melhor seu suporte multilíngue.

```json
// next.config.js
{
  i18n: {
    locales: ['default', 'en', 'zh-CN'],
    defaultLocale: 'default',
  },
}
```

Em seguida, passe o `locale` fornecido pelo Pages Router para o `CinaWalletKitProvider`

```tsx
import { CinaWalletKitProvider, Locale } from '@cinagroup/CinaWalletKit';

export const App = () => {
  const { locale } = useRouter() as { locale: Locale };
  return (
    <CinaWalletKitProvider locale={locale} {...etc}>
      {/* Your App */}
    </CinaWalletKitProvider>
  )
};
```

Você pode consultar um exemplo para o Pages Router [aqui](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next).

O App Router ainda não suporta i18n. Consulte nosso exemplo separado [aqui](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next-app-i18n) para as melhores práticas de implementação com o middleware [`next-intl`](https://github.com/amannn/next-intl).

Recomenda-se que você use as mesmas técnicas para traduzir o conteúdo do seu dApp para obter suporte total à localização. Bibliotecas de localização, como [`i18n-js`](https://github.com/fnando/i18n) e [`next-intl`](https://github.com/amannn/next-intl), e ferramentas de gerenciamento como Crowdin simplificarão esse processo.
