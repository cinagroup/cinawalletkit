# Локалізація

## Налаштування мови вашого dApp з вбудованими перекладами

За замовчуванням, CinaWalletKit підтримує `en-US` локаль для користувачів англійської мови.

Якщо підтримується, CinaWalletKit виявить [переваги користувача](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language) щодо мови та обере відповідні переклади. Розробники завжди можуть змінити мову за замовчуванням.

Щоб вказати мову для ваших користувачів, просто додайте `locale="zh-CN"` як проп у ваш `CinaWalletKitProvider`

```tsx
import { CinaWalletKitProvider } from '@cinagroup/CinaWalletKit';

export const App = () => (
  <CinaWalletKitProvider locale="zh-CN" {...etc}>
    {/* Your App */}
  </CinaWalletKitProvider>
);
```

### Використання з Next.js

Підтримка локалізації в CinaWalletKit ще краще працює з [Sub-path Routing](https://nextjs.org/docs/pages/building-your-application/routing/internationalization#sub-path-routing).

Налаштуйте ваш проект Next.js так, як показано у прикладі нижче, щоб додати додатковий `/locale/` під-шлях до ваших маршрутів, що допоможе пошуковим системам і користувачам краще виявляти вашу багатомовну підтримку.

```json
// next.config.js
{
  i18n: {
    locales: ['default', 'en', 'zh-CN'],
    defaultLocale: 'default',
  },
}
```

Потім передайте параметр `locale`, наданий роутером сторінок, в `CinaWalletKitProvider`

```tsx
import { CinaWalletKitProvider, Locale } from '@cinagroup/CinaWalletKit';

export const App = () => {
  const { locale } = useRouter() as { locale: Locale };
  return (
    <CinaWalletKitProvider locale={locale} {...etc}>
      {/* Your App */}
    </CinaWalletKitProvider>
  )
};
```

Ви можете переглянути приклад для роутера сторінок [тут](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next).

App Router поки не підтримує i18n. Перегляньте наш окремий приклад [тут](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next-app-i18n) для найкращих практик реалізації з [middleware](https://github.com/amannn/next-intl) `next-intl`.

Рекомендується, щоб ви використовували ті самі методики для перекладу вмісту вашого dApp для підтримки повної локалізації. Бібліотеки локалізації, такі як [`i18n-js`](https://github.com/fnando/i18n) та [`next-intl`](https://github.com/amannn/next-intl), а також інструменти управління, такі як Crowdin, спростять цей процес.
