# Користувальницька аутентифікація

## Підключіться до власного бекенду аутентифікації

Хоча CinaWalletKit забезпечує [підтримку Вхід з Ethereum і NextAuth на першокласному рівні,](/docs/authentication#set-up-sign-in-with-ethereum-and-nextauth) ви також можете інтегруватися з користувальницькими бекендами та форматами повідомлень.

### Створення користувальницького адаптера

Спочатку створіть адаптер аутентифікації. Це дозволяє CinaWalletKit створювати/готувати повідомлення та взаємодіяти з вашим бекендом.

Як приклад, ми можемо створити адаптер аутентифікації, який дозволяє нам використовувати [Sign-In з Ethereum](https://login.xyz) за допомогою деяких налаштовуваних точок API, наприклад, [iron-session](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next-siwe-iron-session).

```tsx
import { createAuthenticationAdapter } from '@cinagroup/CinaWalletKit';
import { createSiweMessage } from 'viem/siwe';

const authenticationAdapter = createAuthenticationAdapter({
  getNonce: async () => {
    const response = await fetch('/api/nonce');
    return await response.text();
  },

  createMessage: ({ nonce, address, chainId }) => {
    return createSiweMessage({
      domain: window.location.host,
      address,
      statement: 'Sign in with Ethereum to the app.',
      uri: window.location.origin,
      version: '1',
      chainId,
      nonce,
    });
  },

  verify: async ({ message, signature }) => {
    const verifyRes = await fetch('/api/verify', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ message, signature }),
    });

    return Boolean(verifyRes.ok);
  },

  signOut: async () => {
    await fetch('/api/logout');
  },
});
```

#### Створення повідомлення на сервері

`createMessage` також підтримує повернення `Promise`, що дозволяє вам створювати [EIP-4361](https://eips.ethereum.org/EIPS/eip-4361) повідомлення на сервері. Для більш строгої безпеки, серверний ендпоінт повинен виводити або перевіряти критично важливі поля безпеки, такі як `domain`, `nonce` та мітки часу `issued-at`, замість довіри до значень клієнта. Дивіться [документацію SIWE](https://docs.siwe.xyz/security-considerations#server-side-message-generation) для отримання додаткової інформації.

```tsx
  createMessage: async ({ address, chainId }) => {
    const response = await fetch('/api/siwe/message', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ address, chainId }),
    });

    if (!response.ok) {
      throw new Error('Failed to create message');
    }

    return await response.text();
  },
```

#### Надання стану аутентифікації

Припускаючи, що ваш додаток вже певним чином керує життєвим циклом аутентифікації, ви можете передати поточний статус аутентифікації разом із вашим користувальницьким адаптером до `CinaWalletKitAuthenticationProvider`, обгортаючи ваш існуючий `CinaWalletKitProvider`.

```tsx
import {
  createAuthenticationAdapter,
  CinaWalletKitAuthenticationProvider,
  CinaWalletKitProvider,
} from '@cinagroup/CinaWalletKit';
import { AppProps } from 'next/app';
import { WagmiProvider } from 'wagmi';
import {
  QueryClient,
  QueryClientProvider,
} from '@tanstack/react-query';

const authenticationAdapter = createAuthenticationAdapter({
  /* See above... */
});

const queryClient = new QueryClient();

export default function App({ Component, pageProps }: AppProps) {
  // You'll need to resolve AUTHENTICATION_STATUS here
  // using your application's authentication system.
  // It needs to be either 'loading' (during initial load),
  // 'unauthenticated' or 'authenticated'.

  return (
    <WagmiProvider {...etc}>
      <QueryClientProvider client={queryClient}>
        <CinaWalletKitAuthenticationProvider
          adapter={authenticationAdapter}
          status={AUTHENTICATION_STATUS}
        >
          <CinaWalletKitProvider {...etc}>
            <Component {...pageProps} />
          </CinaWalletKitProvider>
        </CinaWalletKitAuthenticationProvider>
      </QueryClientProvider>
    </WagmiProvider>
  );
}
```

Якщо ви вже створили адаптер для існуючої бібліотеки аутентифікації з відкритим вихідним кодом, будь ласка, розгляньте можливість створення пакета, який можуть використовувати інші!
