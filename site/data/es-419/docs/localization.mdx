# Localización

## Personalizar el idioma de tu dApp con traducciones integradas

Por defecto, CinaWalletKit soporta la configuración regional `en-US` para usuarios de habla inglesa.

Si está disponible, CinaWalletKit detectará el [idioma preferido](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language) del usuario y elegirá las traducciones apropiadas. Los desarrolladores siempre pueden sobrescribir el idioma por defecto.

Para especificar un idioma para tus usuarios, simplemente agrega `locale="zh-CN"` como una propiedad en tu `CinaWalletKitProvider`.

```tsx
import { CinaWalletKitProvider } from '@cinagroup/CinaWalletKit';

export const App = () => (
  <CinaWalletKitProvider locale="zh-CN" {...etc}>
    {/* Your App */}
  </CinaWalletKitProvider>
);
```

### Uso con Next.js

El soporte de localización de CinaWalletKit funciona aún mejor con [Sub-path Routing](https://nextjs.org/docs/pages/building-your-application/routing/internationalization#sub-path-routing).

Configura tu proyecto Next.js como el ejemplo a continuación para agregar una subruta opcional `/locale/` a tus rutas, lo cual ayudará a los motores de búsqueda y usuarios a descubrir mejor tu soporte multilingüe.

```json
// next.config.js
{
  i18n: {
    locales: ['default', 'en', 'zh-CN'],
    defaultLocale: 'default',
  },
}
```

Después, pasa el `locale` proporcionado por el Pages Router al `CinaWalletKitProvider`.

```tsx
import { CinaWalletKitProvider, Locale } from '@cinagroup/CinaWalletKit';

export const App = () => {
  const { locale } = useRouter() as { locale: Locale };
  return (
    <CinaWalletKitProvider locale={locale} {...etc}>
      {/* Your App */}
    </CinaWalletKitProvider>
  )
};
```

Puedes consultar un ejemplo para Pages Router [aquí](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next).

App Router todavía no soporta i18n. Consulta nuestro ejemplo separado [aquí](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next-app-i18n) para las mejores prácticas de implementación con el middleware [`next-intl`](https://github.com/amannn/next-intl).

Se recomienda que utilices las mismas técnicas para traducir el contenido de tu dApp para un soporte de localización completo. Las librerías de localización como [`i18n-js`](https://github.com/fnando/i18n) y [`next-intl`](https://github.com/amannn/next-intl) y herramientas de gestión como Crowdin simplificarán este proceso.
