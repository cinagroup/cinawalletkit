# Lokalisierung

## Anpassung der Sprache Ihrer dApp mit eingebauten Übersetzungen

Standardmäßig unterstützt CinaWalletKit das `en-US`-Locale für englischsprachige Nutzer.

Falls verfügbar, erkennt CinaWalletKit die [bevorzugte Sprache](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language) des Nutzers und wählt die entsprechenden Übersetzungen. Entwickler können die Standard-Sprache immer überschreiben.

Um eine Sprache für Ihre Benutzer festzulegen, fügen Sie einfach `locale="zh-CN"` als Prop in Ihren `CinaWalletKitProvider` ein.

```tsx
import { CinaWalletKitProvider } from '@cinagroup/CinaWalletKit';

export const App = () => (
  <CinaWalletKitProvider locale="zh-CN" {...etc}>
    {/* Your App */}
  </CinaWalletKitProvider>
);
```

### Verwendung mit Next.js

Die Lokalisierungsunterstützung von CinaWalletKit funktioniert noch besser mit [Sub-Pfade-Routing](https://nextjs.org/docs/pages/building-your-application/routing/internationalization#sub-path-routing).

Konfigurieren Sie Ihr Next.js-Projekt wie im folgenden Beispiel, um einen optionalen `/locale/`-Unterpfad zu Ihren Routen hinzuzufügen, was Suchmaschinen und Benutzern hilft, Ihre mehrsprachige Unterstützung besser zu entdecken.

```json
// next.config.js
{
  i18n: {
    locales: ['default', 'en', 'zh-CN'],
    defaultLocale: 'default',
  },
}
```

Übergeben Sie dann das vom Pages-Router bereitgestellte `locale` an den `CinaWalletKitProvider`.

```tsx
import { CinaWalletKitProvider, Locale } from '@cinagroup/CinaWalletKit';

export const App = () => {
  const { locale } = useRouter() as { locale: Locale };
  return (
    <CinaWalletKitProvider locale={locale} {...etc}>
      {/* Your App */}
    </CinaWalletKitProvider>
  )
};
```

Sie können ein Beispiel für den Pages-Router [hier](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next) einsehen.

Der App-Router unterstützt i18n noch nicht. Sehen Sie sich unser separates Beispiel [hier](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next-app-i18n) für die Implementierungsbest-Practices mit [`next-intl`](https://github.com/amannn/next-intl)-Middleware an.

Es wird empfohlen, dieselben Techniken zu verwenden, um den Inhalt Ihrer dApp für vollständige Lokalisierungsunterstützung zu übersetzen. Lokalisierungsbibliotheken wie [`i18n-js`](https://github.com/fnando/i18n) und [`next-intl`](https://github.com/amannn/next-intl) und Management-Tools wie Crowdin werden diesen Prozess vereinfachen.
