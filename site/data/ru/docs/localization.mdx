# Локализация

## Настройка языка вашего dApp с встроенными переводами

По умолчанию, CinaWalletKit поддерживает локаль `en-US` для пользователей английского языка.

При наличии, CinaWalletKit определит [предпочтительный язык](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language) пользователя и выберет соответствующие переводы. Разработчики всегда могут изменить язык по умолчанию.

Чтобы указать язык для ваших пользователей, просто добавьте `locale="zh-CN"` как пропс в вашем `CinaWalletKitProvider`.

```tsx
import { CinaWalletKitProvider } from '@cinagroup/CinaWalletKit';

export const App = () => (
  <CinaWalletKitProvider locale="zh-CN" {...etc}>
    {/* Your App */}
  </CinaWalletKitProvider>
);
```

### Использование с Next.js

Поддержка локализации в CinaWalletKit работает еще лучше с [маршрутизацией по под-путям](https://nextjs.org/docs/pages/building-your-application/routing/internationalization#sub-path-routing).

Настройте ваш проект Next.js, как показано в примере ниже, чтобы добавить необязательный подпуть `/locale/` к вашим маршрутам, что поможет пользователям и поисковым системам лучше обнаружить вашу многоязыковую поддержку.

```json
// next.config.js
{
  i18n: {
    locales: ['default', 'en', 'zh-CN'],
    defaultLocale: 'default',
  },
}
```

Затем передайте `locale`, предоставленное маршрутизатором Pages, в `CinaWalletKitProvider`.

```tsx
import { CinaWalletKitProvider, Locale } from '@cinagroup/CinaWalletKit';

export const App = () => {
  const { locale } = useRouter() as { locale: Locale };
  return (
    <CinaWalletKitProvider locale={locale} {...etc}>
      {/* Your App */}
    </CinaWalletKitProvider>
  )
};
```

Вы можете посмотреть пример для маршрутизатора Pages [здесь](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next).

Маршрутизатор App пока не поддерживает i18n. Ссылка на отдельный пример [здесь](https://github.com/rainbow-me/CinaWalletKit/tree/main/examples/with-next-app-i18n) для лучших практик реализации с [`next-intl`](https://github.com/amannn/next-intl) middleware.

Рекомендуется использовать те же методы для перевода контента вашего dApp для полной поддержки локализации. Такие библиотеки для локализации, как [`i18n-js`](https://github.com/fnando/i18n) и [`next-intl`](https://github.com/amannn/next-intl), и инструменты управления, такие как Crowdin, упростят этот процесс.
