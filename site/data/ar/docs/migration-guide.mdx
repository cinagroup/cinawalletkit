# دليل الترحيل

## ترحيل CinaWalletKit

### تغييرات جوهرية في الإصدار 2.x.x

وصلت الاعتمادات النظيرة [wagmi](https://wagmi.sh) و[viem](https://viem.sh) إلى `2.x.x` مع تغييرات مهمة.

اتبع الخطوات أدناه للهجرة.

**1. قم بترقية CinaWalletKit و`wagmi` و`viem` إلى أحدث الإصدارات**

```bash
npm i @cinagroup/CinaWalletKit@2 wagmi@2 viem@2.x
```

**2. قم بتثبيت التبعية النظيرة `@tanstack/react-query`**

مع Wagmi v2، أصبح [TanStack Query](https://tanstack.com/query/v5/docs/react/overview) الآن اعتماداً نظيراً مطلوبًا.

قم بتثبيته باستخدام الأمر التالي:

```bash
npm i @tanstack/react-query
```

**3. قم بترقية إعدادات CinaWalletKit وWagmi**

```diff
  import '@cinagroup/CinaWalletKit/styles.css'

+ import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
- import { createPublicClient, http } from 'viem'

+ import { WagmiProvider, http } from 'wagmi'
- import { configureChains, createConfig } from 'wagmi'
  import { mainnet } from 'wagmi/chains'
  import { CinaWalletKitProvider } from '@cinagroup/CinaWalletKit'
- import { getDefaultWallets, connectorsForWallets } from '@cinagroup/CinaWalletKit'
+ import { getDefaultConfig } from '@cinagroup/CinaWalletKit'

  /* getDefaultWallets is now optional */
- const { wallets } = getDefaultWallets({
-   appName: 'CinaWalletKit demo',
-   projectId: 'YOUR_PROJECT_ID',
-   chains,
- })

  /* connectorsForWallets is now optional */
- const connectors = connectorsForWallets([...wallets])

- const { chains, publicClient } = configureChains( 
-   [mainnet, sepolia], 
-   [publicProvider(), publicProvider()],
- )

- const config = createConfig({
-   autoConnect: true,
-   publicClient,
- })

  /* New API that includes Wagmi's createConfig and replaces getDefaultWallets and connectorsForWallets */
+ const config = getDefaultConfig({
+   appName: 'CinaWalletKit demo',
+   projectId: 'YOUR_PROJECT_ID',
+   chains: [mainnet],
+   transports: {
+     [mainnet.id]: http(),
+   },
+ })

+         <CinaWalletKitProvider>

```

**4. تحقق من التغييرات الجذرية في `wagmi` و`viem`**

إذا كنت تستخدم الأساليب والوظائف `wagmi` و`viem` في تطبيقك، فستحتاج إلى اتباع إرشادات الهجرة للإصدار v2:

- [دليل هجرة Wagmi v2](https://wagmi.sh/react/guides/migrate-from-v1-to-v2)
- [التغييرات الجذرية في Viem v2](https://viem.sh/docs/migration-guide#2xx-breaking-changes)

#### تغييرات CinaWalletKit

**1. تحسين السلوك لمحافظ EIP-6963**

المحافظ التي تدعم معيار الاتصال الجديد EIP-6963 (بما في ذلك Rainbow وMetaMask وغيرها) ستظهر الآن تلقائيًا في قسم `Installed` أثناء تجربة الاتصال بالمحفظة. يضمن ذلك أن المستخدمين يمكن أن يجدوا محافظهم المفضلة دائمًا ويتصلوا بتطبيقات dApps بدون تعارضات أو أزرار احتياطية.

لا يزال لدى المطورين السيطرة الكاملة على قائمة المحافظ المخصصة لتأكيد المحافظ المفضلة للمستخدمين النهائيين. يُفضل أن تستمر في تضمين `injectedWallet` و`walletConnectWallet` في قائمتك لدعم جميع المنصات.

**2. إعداد Wagmi مع getDefaultConfig**

يُبسط هذا API الجديد تجربة الإعداد ويستبدل الحاجة لاستخدام `createConfig` في Wagmi مباشرةً. تم تبسيط إعداد السلسلة، بما في ذلك توفر مقدمي الخدمة العامين لـ `transports`.

سيتم تضمين قائمة المحافظ الافتراضية تلقائيًا، مما يلغي الحاجة لاستخدام `getDefaultWallets` و`connectorsForWallets`.

يمكنك إنشاء قائمة محافظ مخصصة عن طريق تمرير موصلات محافظ مستوردة أو مخصصة إلى `wallets`. لم يعد من الضروري إنشاء موصلات المحافظ وتمرير `projectId` و`chains`.

```
const config = getDefaultConfig({
  appName: 'CinaWalletKit demo',
  projectId: 'YOUR_PROJECT_ID',
  chains: [mainnet],
  wallets: [rainbowWallet], /* optional custom wallet list */
  /* Wagmi createConfig options including `transports` are also accepted */
})
```

**3. CinaWalletKitProvider**

لم تعد بحاجة إلى تمرير `chains` إلى `<CinaWalletKitProvider>`.

```diff
- <CinaWalletKitProvider chains={chains}>
+ <CinaWalletKitProvider>
```

**4. السلاسل المخصصة**

تغيّر نوع `Chain` وفقًا لـ Wagmi v2، ويواصل دعم بيانات CinaWalletKit التعريفية مثل `iconUrl` و`iconBackground`.

```diff
+ import { Chain } from '@cinagroup/CinaWalletKit'

  const avalanche = {
    id: 43_114,
    name: 'Avalanche',
    iconUrl: 'https://s2.coinmarketcap.com/static/img/coins/64x64/5805.png',
    iconBackground: '#fff',
    nativeCurrency: { name: 'Avalanche', symbol: 'AVAX', decimals: 18 },
    rpcUrls: {
      default: { http: ['https://api.avax.network/ext/bc/C/rpc'] },
    },
    blockExplorers: {
      default: { name: 'SnowTrace', url: 'https://snowtrace.io' },
    },
    contracts: {
      multicall3: {
        address: '0xca11bde05977b3631167028862be2a173976ca11',
        blockCreated: 11_907_934,
      },
    },
  } as const satisfies Chain
```

مثال مع `getDefaultConfig`:

```diff
const config = getDefaultConfig({
+ chains: [
+   avalanche, /* custom chain */
+   {
+     ...mainnet,
+     iconBackground: '#000',
+     iconUrl: 'https://example.com/icons/ethereum.png',
+   }, /* metadata overides */
+ ],
});
```

مثال مع `createConfig`:

```diff
+ import { Chain } from '@cinagroup/CinaWalletKit'
+ const chains: readonly [Chain, ...Chain[]] = [
+   {
+     ...mainnet,
+     iconBackground: '#000',
+     iconUrl: 'https://example.com/icons/ethereum.png',
+   },
+ ];
  const config = createConfig({
    chains,
    transports: {
      [mainnet.id]: http(),
    },
  })
```

**5. المحافظ المخصصة**

طرأت تغييرات كبيرة على موصلات محافظ CinaWalletKit لدعم Wagmi v2. ارجع إلى [الوثائق المحدثة](https://www.CinaWalletKit.com/docs/custom-wallets) و[مثال الموصل](https://github.com/rainbow-me/CinaWalletKit/blob/main/packages/CinaWalletKit/src/wallets/walletConnectors/rainbowWallet/rainbowWallet.ts) لترقية أي موصلات محافظ مخصصة في dApp الخاص بك.

يدعم موصلين المحافظ الآن أيضًا معيار EIP-6963 مع خاصية `rdns`. تأكد من ملئها لمنع الإشارات المكررة إلى المحافظ الداعمة لـ EIP-6963 في قائمة المحافظ الخاصة بك.

### تغييرات جوهرية في الإصدار 0.12.x

تم تحديث التبعية النظيرة لـ wagmi إلى `0.12.x`.

اعتمدت CinaWalletKit على موصل `WalletConnectLegacyConnector` في wagmi لدعم WalletConnect v1 المستمر. سيتم إصدار دعم للـ WalletConnect v2 وموصل `WalletConnectConnector` قريبًا كإصدار ترقيعي، دون تغييرات جذرية.

سيتم نقل محافظ تلقائيًا في الإصدارات المستقبلية.

يجب على كل dApp الآن توفير [WalletConnect Cloud](https://cloud.walletconnect.com/) `projectId` لتمكين WalletConnect v2. يجب إنجاز ذلك قبل إيقاف تشغيل خوادم الجسر لـ WalletConnect v1 في 28 يونيو 2023. سيفضل CinaWalletKit بشكل هادئ v1 لجميع المحافظ إذا لم يتم تحديد `projectId`.

اتبع الخطوات أدناه للهجرة.

#### 1. قم بترقية CinaWalletKit و`wagmi` إلى أحدث نسخة

```bash
npm i @cinagroup/CinaWalletKit@^0.12.0 wagmi@^0.12.0
```

#### 2. توفير مشروع WalletConnect Cloud projectId

كل dApp يعتمد على WalletConnect يحتاج الآن للحصول على `projectId` من [WalletConnect Cloud](https://cloud.walletconnect.com/). هذا عمل مجاني تمامًا ولا يستغرق سوى بضع دقائق.

قدم `projectId` إلى `getDefaultWallets` وموصلات محافظ CinaWalletKit الفردية كالتالي:

```ts
const projectId = 'YOUR_PROJECT_ID';

const { wallets } = getDefaultWallets({
  appName: 'My CinaWalletKit App',
  projectId,
  chains,
});

const connectors = connectorsForWallets([
  ...wallets,
  {
    groupName: 'Other',
    wallets: [
      readyWallet({ projectId, chains }),
      trustWallet({ projectId, chains }),
      ledgerWallet({ projectId, chains }),
    ],
  },
]);
```

### تغييرات جوهرية في الإصدار 0.2.x

تم تحديث التبعية النظيرة لـ CinaWalletKit إلى `^0.4`.

اتبع الخطوات أدناه للهجرة.

#### 1. قم بترقية التبعية

قم بترقية CinaWalletKit وwagmi إلى أحدث نسخة

```bash
npm i @cinagroup/CinaWalletKit@^0.2.0 wagmi@^0.4.2
```

#### 2. استبدال استيراد configureChains

استورد `configureChains` من wagmi بدلاً من CinaWalletKit:

```diff
- import { configureChains } from '@cinagroup/CinaWalletKit';
+ import { configureChains } from 'wagmi';
```

#### 3. ترحيل المزودين

لم يعد CinaWalletKit يصدر API لـ `apiProvider`. استبدله بالمزود المرغوب فيه من wagmi.

```diff
- import { apiProvider } from '@cinagroup/CinaWalletKit';
```

#### Alchemy

استورد `alchemyProvider` من `wagmi/providers/alchemy`.

```diff
+ import { alchemyProvider } from 'wagmi/providers/alchemy';

const { chains, provider } = configureChains(
  [chain.mainnet, chain.polygon, chain.optimism, chain.arbitrum],
- [apiProvider.alchemy(process.env.ALCHEMY_ID)]
+ [alchemyProvider({ alchemyId: process.env.ALCHEMY_ID })]
);
```

#### Infura

استورد `infuraProvider` من `wagmi/providers/infura`.

```diff
+import { infuraProvider } from 'wagmi/providers/infura';

const { chains, provider } = configureChains(
  [chain.mainnet, chain.polygon, chain.optimism, chain.arbitrum],
- [apiProvider.infura(process.env.INFURA_ID)]
+ [infuraProvider({ infuraId: process.env.INFURA_ID })]
);
```

#### JSON RPC

استورد `jsonRpcProvider` من `wagmi/providers/jsonRpc`.

```diff
+ import { jsonRpcProvider } from 'wagmi/providers/jsonRpc';

const { chains, provider } = configureChains(
  [chain.mainnet, chain.polygon],
  [
-   apiProvider.jsonRpc(chain => ({
-     rpcUrl: `https://${chain.id}.example.com`,
-   })),
+   jsonRpcProvider({
+     rpc: chain => ({
+       http: `https://${chain.id}.example.com`,
+     }),
+   }),
  ]
);
```

#### مزود عام

استورد `publicProvider` من `wagmi/providers/public`.

```diff
+ import { publicProvider } from 'wagmi/providers/public';

const { chains, provider } = configureChains(
  [chain.mainnet, chain.polygon],
- [apiProvider.fallback()]
+ [publicProvider()]
);
```

#### 4. أعد تسمية المزود في wagmi

أعد تسمية `WagmiProvider` إلى `WagmiConfig`.

```diff

- WagmiProvider
+ WagmiConfig
} from 'wagmi'

const App = () => {
  return (
-   <WagmiProvider client={wagmiClient}>...</WagmiProvider>
+   <WagmiConfig client={wagmiClient}>...</WagmiConfig>
  );
};
```
